# Scenario 2 — SMB Brute Force Detection Investigation
 
## Overview
 
This investigation documents the detection and analysis of a credential access attack performed from the Kali Linux attacker machine against the Windows victim endpoint within the SOC Home Lab environment.
 
The objective was to validate Elastic SIEM visibility, alert generation, and SOC analyst investigation procedures for detecting SMB brute force / password guessing activity.

## Attack Description
 
A brute force attack was launched from the Kali Linux attacker machine against the Windows victim endpoint using a Python Impacket-based SMB brute force script targeting the administrator account over the SMB protocol.
 
The objective of the attack was to guess valid administrator credentials in order to gain unauthorized access to the target system.
 
Elastic SIEM collected Windows Security Event ID 4625 (failed logon) events and generated a High-severity alert based on the abnormal volume of authentication failures from a single source.
 
---
 
## What I Found in the Logs
 
Investigation revealed that source host **10.0.0.5** conducted a brute force attack using the SMB protocol against **10.0.0.4**, targeting the built-in administrator account.
 
A total of **20 events** were generated by the SIEM during the attack window.
 
The following port was targeted during the attack:
 
| Port | Service |
| ---- | ------- |
| 445  | SMB     |
 
The attack pattern showed repeated failed logon attempts within seconds, followed by an account lockout — consistent with automated credential guessing behavior.

## True Positive or False Positive
 
**Classification:** True Positive (TP)
 
### Why?
 
Brute force is a high-severity attack. The attacker specifically targeted the administrator account, which has elevated privileges and full system access. The activity originated from an internal host (10.0.0.5), generated 20 failed logon events within seconds, and ultimately caused an account lockout — confirming automated credential guessing activity.
 
### Evidence — Detection Alert Generated
 
The custom Elastic Security detection rule successfully generated a High severity alert based on the abnormal number of failed logon events.

## Actions Taken
 
* Identified source host 10.0.0.5 as an unauthorized internal host.
* Monitored for follow-up activity on port 445.
* Escalated to Tier 2 analyst for further investigation.
* Recommended blocking 10.0.0.5 at the network firewall.
---
 
## Lessons Learned
 
* Event ID 4625 requires the Security channel to be explicitly added to the Fleet integration for Elastic to capture it.
* The Audit Logon Failure policy must be enabled via `auditpol` on the Windows victim for these events to generate at all.
* The administrator account must be active (not disabled) for authentication failures against it to be logged.
* Threshold-based detection rules are effective for identifying brute force activity even at low event volumes.
---
 
## Conclusion
 
An SMB brute force attack originating from the Kali Linux attacker machine, targeting the administrator account on the Windows victim host, was successfully detected and investigated using Elastic SIEM.
 
The attack generated 20 Event ID 4625 (failed logon) events and triggered a High severity alert from a custom threshold-based detection rule. The incident was classified as a True Positive and mapped to MITRE ATT&CK techniques T1110 (Brute Force) and T1110.001 (Password Guessing), demonstrating the effectiveness of the SOC Home Lab for detection engineering and incident investigation practice.
